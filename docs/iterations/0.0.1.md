# 0.0.1 — MVP 合同管理系统

## 范围
- 登录认证（JWT）+ 演示账号
- 用户管理（admin CRUD, 角色分配）
- 合同 CRUD + 状态流转（草稿→待签→已签→终止/过期）
- 附件上传下载（pdf/word 类型校验, ≤10MB）
- 操作审计日志
- 公网部署到 http://120.24.117.67/projects/contract-manager-eval/

## 业务模型
详见 events e000002 `requirement_defined`

- User: username, password_hash, display_name, email, role(admin/manager/viewer), status(active/disabled)
- Contract: contract_no, title, party_a, party_b, amount, status(draft/pending/signed/terminated/expired), sign_date, start_date, end_date, description, created_by FK
- Attachment: contract_id FK, filename, original_name, file_type(pdf/word), file_size, storage_path, uploaded_by FK
- AuditLog: user_id FK, action, entity_type, entity_id, detail

### 关系
- User 1:N Contract (created_by)
- Contract 1:N Attachment (contract_id)
- User 1:N AuditLog (user_id)

### 状态机
```
Contract.status: draft → pending → signed → terminated
                                         → expired
```
- draft → pending: 提交审批
- pending → signed: 签署
- signed → terminated: 终止
- signed → expired: 过期（按 end_date 自动判定）
- draft → terminated: 撤回（草稿可直接终止）
- pending → draft: 退回

## 产品流程
1. 登录 → 用户名+密码 → bcrypt 验证 → JWT 签发 → 跳转合同列表
2. 合同管理 → 列表页（分状态筛选、搜索合同编号/标题）→ 新建/编辑表单 → 详情页
3. 合同状态流转 → 操作按钮根据当前状态和用户角色动态显示
4. 附件管理 → 上传（客户端校验扩展名 + 服务端校验 magic bytes）→ 下载 → 删除
5. 用户管理 → admin 角色可见 → 创建/编辑/启禁用用户
6. 异常处理 → 未登录重定向 /login、无权限 403、文件类型不符拒绝、资源不存在 404

**API 设计：**

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| POST | /api/auth/login | No | 登录，返回 JWT |
| GET | /api/users | admin | 用户列表 |
| POST | /api/users | admin | 创建用户 |
| PUT | /api/users/{id} | admin | 编辑用户 |
| GET | /api/contracts | Any | 合同列表（支持 status/q 参数筛选） |
| POST | /api/contracts | manager+ | 创建合同 |
| GET | /api/contracts/{id} | Any | 合同详情 |
| PUT | /api/contracts/{id} | manager+ | 编辑合同 |
| DELETE | /api/contracts/{id} | admin | 删除合同 |
| POST | /api/contracts/{id}/transition | manager+ | 状态流转 |
| POST | /api/contracts/{id}/attachments | manager+ | 上传附件 |
| GET | /api/contracts/{id}/attachments/{aid} | Any | 下载附件 |
| DELETE | /api/contracts/{id}/attachments/{aid} | manager+ | 删除附件 |

**附件校验规则：**
- 扩展名白名单：`.pdf`, `.doc`, `.docx`
- Magic bytes 校验：
  - PDF: 文件头 `%PDF-`（bytes 0-4）
  - DOCX: 文件头 `PK\x03\x04`（ZIP 格式）
  - DOC: 文件头 `\xD0\xCF\x11\xE0`（OLE2 格式）
- 文件大小：≤ 10MB (10 * 1024 * 1024 bytes)
- 存储路径：`uploads/{contract_id}/{uuid}_{original_name}`

### 前端：Jinja2 SSR + 静态 CSS/JS

- 服务端模板渲染，无 SPA 框架
- 静态资源路径：`/projects/contract-manager-eval/static/{css,js}/*?v=0.0.1`
- 所有 HTML 响应头带 `Cache-Control: no-cache`
- 导航栏根据用户角色动态显示菜单项
- 表单客户端校验 + 服务端错误展示

### 数据库：SQLite

- 文件：`data/contract_manager.db`
- 初始化脚本 `scripts/init_db.py` 创建所有表和种子数据
- 种子数据（bcrypt 哈希密码）：
  - admin / admin123 → role=admin
  - manager / manager123 → role=manager
  - viewer / viewer123 → role=viewer

### 认证：JWT

- Token 有效期：8 小时
- 存储方式：Cookie（HttpOnly for API）或 localStorage（前端用）
- 实现方式：前端用 Authorization Bearer header，服务端 Jinja2 用 Cookie

## 风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| SQLite 并发限制 | 多用户同时写入可能锁等待 | 演示项目，并发低；WAL 模式 |
| Jinja2 SSR 无前端路由 | 页面刷新丢失状态 | 服务端维护 session/JWT Cookie |
| 文件上传安全 | 恶意文件上传风险 | magic bytes + 扩展名双重校验 |
| 公开仓库密钥泄露 | JWT_SECRET 等泄漏 | .env 不入库，.env.example 仅模板 |
| Nginx base path 配置 | 静态资源路径错误 | 统一使用 url_for 生成路径 |
